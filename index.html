如何用 Next.js 构建个人博客

作者：老六
日期：2025-08-20

前言

随着前端技术的发展，越来越多的开发者选择使用 Next.js 来搭建个人博客。本文将分享使用 Next.js 快速搭建博客的完整流程和一些实用经验。

1. 项目初始化

首先，使用官方脚手架创建项目：

npx create-next-app my-blog

2. 页面路由与布局

Next.js 支持文件式路由，只需在 pages 或 app 目录下添加对应的文件即可自动生成页面。推荐使用 app 目录实现更灵活的布局。

3. 数据管理

可以使用 getStaticProps 或 getServerSideProps 获取数据，也可以集成 CMS 或数据库。对于个人博客，建议将文章内容以 Markdown 文件存储，便于管理和编辑。

4. 部署上线

推荐使用 Vercel 平台部署 Next.js 项目，只需一键即可自动构建和上线。

5. 常见问题与解决方案

  • 样式丢失：检查 CSS 文件是否正确引入。
  • 路由404：确认页面文件命名和路径是否正确。

总结

Next.js 提供了强大的功能和极佳的开发体验，非常适合个人博客和内容网站的搭建。希望本文能帮助你快速上手并顺利上线自己的博客！

ThinkPHP 框架开发入门与实战

作者：老六
日期：2025-09-11

什么是 ThinkPHP？

ThinkPHP 是国内流行的 PHP 开发框架，具有高效、简洁、易扩展等特点，适合快速开发各类 Web 应用。

1. 快速安装

推荐使用 Composer 安装：

composer create-project topthink/think tpapp

2. 目录结构简介

tpapp/
├─ app/         # 应用目录
├─ public/      # WEB 入口目录
├─ config/      # 配置文件目录
├─ route/       # 路由定义目录
├─ vendor/      # 第三方库

3. 创建控制器

在 app/controller 目录下新建 Index.php：

<?php
namespace app\controller;

class Index
{
    public function index()
    {
        return '欢迎使用 ThinkPHP！';
    }
}

4. 路由配置

在 route/app.php 添加路由：

use think\facade\Route;

Route::get('/', 'Index/index');

作者：老六
日期：2025-09-18

2. 目录结构简介

assets   前端资源文件夹，用于管理css js等前端资源文件等

commands   包含命令行命令，文件为控制器文件

config 应用的配置文件

controllers 控制器文件

mail  （应用的前后台和命令行的与邮件相关的布局文件等）

modles 模型文件

runtime 程序运行时生成的临时文件

tests     用于各种测试程序的测试类文件

vendor  就是各种第三方的程序。 这是Composer安装的其他程序的存放目录，包含Yii框架本身。

如果你向composer.json 目录增加了新的安装的程序，下次调用Composer的时候， 就会把新安装的目录也安装在这个 vendor 下面。

views 视图文件

web  入口文件的存放目录，对于Web服务器可以访问的目录。其他所有的目录不对Web用户显示。

requirements.php  测试环境是否符合yii框架的要求

yii 文件 它没有扩展名，我们不好知道其具体类型。 但是从文件内容的第一行 #!/usr/bin/env php 来看，这是一个bash脚本。 第一行在告诉bash，也在告诉我们，这是一个使用PHP运行的脚本。

对于Windows系统而言，命令行的入口脚本仍然是 yii ，但是命令行下无法直接运行。所以Yii为我们准备了一个 yii.bat 。这个文件会以 php yii 形式调用PHP来运行入口脚本 。

yii2的入口文件 web/index.php

<?php
defined('YII_DEBUG') or define('YII_DEBUG', true);
/**
 * 在调试模式下执,默认情况下YII_DEBUG为ture.如果定义了YII_DEBUG,则后面的语句将不执行.
 * 如果定义了 YII_DEBUG ,那么表示当前为调试状态,应用在运行过程中,会有一些调试信息的输出.
 */
defined('YII_ENV') or define('YII_ENV', 'dev');
/**
 * dev prod 是安装后默认的两个环境，分别表示开发环境和最终的成品环境。默认情况YII_ENV为prod 表示产品环境。
 * 如果定义了YII_ENV应用将运行于 dev 环境。此外还有一个 test 环境，表示测试环境。
 * YII_ENV 的 dev prod test 三种环境， 会分别使 YII_ENV_DEV YII_ENV_PROD YII_ENV_TEST 的值为 true 。
 * 在应用的配置中，可以对不同环境作出不同的配置。
 */
require(__DIR__ . '/../vendor/autoload.php');
require(__DIR__ . '/../vendor/yiisoft/yii2/Yii.php');
/**
 *  注册Composer自动加载器，yii.php包含Yii的类文件。
 */

$config = require(__DIR__ . '/../config/web.php');  //引入配置文件。

(new yii\web\Application($config))->run();          // 实例化Application类，通过run（）调用。
